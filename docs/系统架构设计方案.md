# AI股票分析助手系统架构设计方案

## 1. 整体架构

### 1.1 架构概述
- **架构风格**：前后端分离架构
- **前端**：Vue3 + JavaScript
- **后端**：Python + Flask
- **数据库**：MySQL + Redis
- **AI模型**：CrewAI（主要）+ 简单机器学习
- **数据获取**：API接口 + 网络爬虫
- **部署**：暂时本地部署，跑通代码

### 1.2 模块划分
- **股票分析模块**：提供股票基本面和技术面分析
- **持仓管理模块**：管理用户的股票持仓
- **关注列表监控模块**：实时监控用户关注的股票，设置触发条件并发送通知
- **用户管理模块**：处理用户认证和权限
- **数据采集模块**：从多个来源获取股票数据
- **AI模型模块**：提供股票分析、持仓管理和关注列表监控功能
- **分析报告管理模块**：管理历史分析报告，支持导出和分享
- **交易记录管理模块**：管理用户交易记录，支持统计分析
- **市场板块分析模块**：分析市场板块表现和相关性
- **数据导入/导出模块**：支持数据的导入和导出
- **通知中心模块**：管理系统通知和提醒
- **帮助中心模块**：提供系统使用指南和文档
- **用户反馈模块**：收集和处理用户反馈
- **AI模型管理模块**：管理AI模型的配置和性能

## 2. 数据流程架构

### 2.1 前端请求层
- **用户交互**：用户通过Vue3前端界面发起请求
- **请求处理**：使用Axios等HTTP客户端库向Flask后端发送API请求
- **数据展示**：接收后端返回的数据，通过ECharts等图表库进行可视化展示

### 2.2 后端API层
- **请求路由**：Flask应用通过蓝图(Blueprint)组织API路由
- **参数验证**：使用Flask-WTF等库验证请求参数的合法性
- **业务逻辑调用**：根据请求类型调用相应的Flow
- **响应返回**：将处理结果以JSON格式返回给前端

### 2.3 数据采集层（后端实现）
- **API接口**：通过AkShare、Baostock、Yahoo Finance API、新浪财经API等获取股票价格、财务数据
- **网络爬虫**：使用Scrapy等框架爬取新闻、行业数据等非结构化信息（单独讨论）
- **数据调度**：通过Celery等任务队列实现定时数据采集和更新

### 2.4 数据处理层
- **数据清洗**：处理缺失值、异常值，确保数据质量
- **特征提取**：从原始数据中提取技术指标、基本面指标等特征
- **数据转换**：将数据转换为适合AI模型输入的格式
- **数据存储**：
  - MySQL：存储结构化数据，如用户信息、股票基本信息、持仓记录、分析结果等
  - Redis：缓存热点数据，如实时股票价格、API调用结果等，以及用于模型间的消息传递

### 2.5 AI模型层
- **Flow设计**：
  - **StockAnalysisFlow**：股票分析流程，管理股票分析的完整生命周期
  - **PortfolioAnalysisFlow**：持仓管理流程，管理投资组合分析的完整生命周期
  - **WatchlistAnalysisFlow**：关注列表监控流程，管理实时监控和通知的完整生命周期
- **CrewAI多Agent系统**：
  - **StockAnalysisCrew**：股票分析团队，包括首席分析师、行业研究员、财务分析师、技术分析师、舆情分析师
  - **PortfolioAnalysisCrew**：持仓分析团队，包括投资组合管理器、风险分析师、资产配置分析师、绩效分析师、再平衡分析师
  - **WatchlistAnalysisCrew**：关注列表监控团队，包括关注列表管理器、自然语言处理专家、实时数据分析师、逻辑评估师、通知专家
- **模型调用**：通过Flask API调用CrewAI模型
- **模型协作**：通过Redis发布/订阅模式实现模型间的实时通信

### 2.6 结果处理层
- **分析结果整合**：将AI模型的分析结果与原始数据整合
- **结果存储**：将分析结果存储到MySQL中，便于后续查询
- **结果缓存**：将频繁访问的分析结果缓存到Redis中
- **结果格式化**：将结果格式化为前端需要的格式
- **通知发送**：当关注列表监控触发条件时，发送通知

### 2.7 安全与监控层
- **认证授权**：使用JWT等方式实现用户认证和权限控制
- **数据加密**：对敏感数据进行加密存储和传输
- **请求监控**：记录API请求日志，监控系统运行状态
- **异常处理**：捕获和处理系统异常，确保系统稳定运行
- **Flow监控**：使用CrewAI Tracing监控Flow执行
- **模型监控**：监控模型性能和预测准确性
- **数据共享监控**：监控模型间的数据传递和协同

## 3. 技术实现细节

### 3.1 前端实现
- **Vue3组件**：使用Composition API组织组件逻辑
- **状态管理**：使用Pinia管理全局状态
- **路由**：使用Vue Router实现页面导航
- **UI库**：使用Element Plus等UI组件库构建界面
- **数据可视化**：使用ECharts实现数据可视化，Vue-chartjs作为补充
- **HTTP客户端**：使用Axios处理HTTP请求，与后端API通信
- **构建工具**：使用Vite作为构建工具
- **CSS预处理器**：使用SCSS
- **代码规范**：使用ESLint + Prettier
- **测试框架**：使用Vitest

### 3.2 后端实现
- **Flask扩展**：使用Flask-RESTful、Flask-SQLAlchemy等扩展
- **数据库连接**：使用SQLAlchemy ORM操作MySQL
- **缓存**：使用Redis-py操作Redis
- **任务队列**：使用Celery处理异步任务

### 3.3 数据采集实现
- **API客户端**：使用Requests、httpx等库调用第三方API
- **爬虫**：使用Scrapy或BeautifulSoup进行网页爬取
- **数据解析**：使用Pandas处理和分析数据

### 3.4 AI模型实现
- **CrewAI配置**：
  - 定义三个主要Crew：StockAnalysisCrew、PortfolioAnalysisCrew、WatchlistAnalysisCrew
  - 为每个Crew配置专业Agent和任务
  - 使用Pydantic模型定义结构化输出
- **工具实现**：
  - 数据查询工具：MySQLQueryTool
  - 网络搜索工具：SerperDevTool
  - 分析工具：FinancialAnalysisTool、TechnicalAnalysisTool等
  - 自然语言处理工具：NLPTool、LogicParserTool
  - 实时数据工具：RealTimeDataTool
  - 通知工具：NotificationTool
  - 数据共享工具：RedisClientTool、DataSyncTool
- **控制原语**：
  - 任务护栏：验证分析结果的合理性和报告质量
  - 结构化输出：使用Pydantic模型确保输出格式一致
  - LLM钩子：监控LLM调用和处理响应
- **模型部署**：
  - 将Flow集成到Flask应用中
  - 使用kickoff_async处理长时间运行的任务
  - 使用@persist装饰器保存Flow状态
- **模型协作**：
  - 使用Redis发布/订阅模式实现模型间的实时通信
  - 使用MySQL存储分析结果
  - 统一数据结构：所有模型使用相同的Pydantic模型定义

## 4. 系统扩展性考虑

1. **模块化设计**：各层之间通过明确的接口通信，便于单独升级和扩展
2. **微服务预留**：虽然当前采用单体应用架构，但预留微服务接口，便于未来拆分
3. **数据源扩展**：设计可插拔的数据源接口，便于添加新的数据源
4. **模型扩展**：支持添加新的AI模型和分析方法
5. **前端扩展**：使用Vue3的组件化设计，便于添加新的功能模块

## 5. 性能优化策略

1. **数据缓存**：使用Redis缓存热点数据，减少数据库查询
2. **异步处理**：使用Celery处理耗时的数据采集和模型训练任务
3. **数据库优化**：合理设计索引，优化查询语句
4. **前端优化**：使用Vue3的虚拟DOM、代码分割等特性提高前端性能
5. **API优化**：实现API请求合并、分页等机制，减少网络传输量
