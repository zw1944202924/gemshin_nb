# AI分析模型设计方案（最终版，简化版，去掉CrewAI）
## 1. 整体架构
### 1.1 设计理念
- **极简原则**：去掉复杂的多智能体架构，直接对接大模型，开发速度快，维护成本低
- **效果优先**：针对A股股票分析场景优化Prompt，确保分析准确，符合国内市场特点
- **兼容原设计**：输出格式和原CrewAI方案完全一致，不影响上层接口和前端展示
### 1.2 系统架构
```
前端请求 → FastAPI后端 → 数据采集模块 → 数据预处理 → 结构化Prompt组装 → 大模型API调用 → 结果解析 → 结构化输出 → 存储/返回前端
```
## 2. 数据采集和预处理（和原设计一致）
### 2.1 数据来源
- **A股基础数据**：AkShare开源接口，免费获取股票代码、名称、价格、K线、财务指标、行业信息等
- **新闻舆情数据**：爬取东方财富、同花顺等财经网站的相关股票新闻、公告
- **市场行情数据**：实时行情、板块涨跌、大盘走势等数据
### 2.2 数据预处理
1. **数据清洗**：过滤无效数据、异常值，补全缺失数据
2. **特征提取**：计算常用技术指标（PE、PB、ROE、毛利率、净利润增长率、MACD、KDJ、RSI等）
3. **结构化整理**：将所有数据整理成大模型容易理解的结构化格式，减少大模型理解成本，提高准确率
## 3. 核心分析流程
### 3.1 股票分析流程
#### 3.1.1 输入参数
- 股票代码、分析时间范围（默认近1年）、分析类型（基本面/技术面/综合分析）
#### 3.1.2 处理步骤
1. **数据采集**：根据股票代码采集对应的基础数据、财务数据、技术指标、近期新闻
2. **Prompt组装**：将结构化数据和分析要求组装成优化后的Prompt，示例：
```
你是专业的A股股票分析师，请基于以下信息对{股票名称}({股票代码})进行{分析类型}分析：
1. 股票基本信息：{基本信息}
2. 财务数据（近4季度）：{财务数据}
3. 技术指标：{技术指标}
4. 近期新闻（近1个月）：{新闻列表}
要求：
- 输出结构化的分析结果，包含：行业评分、财务评分、技术评分、舆情评分、综合评分（满分100）
- 分析结论要客观，符合A股市场特点，不要有套话
- 明确给出投资建议：买入/持有/卖出，以及对应的理由
- 列出主要的风险提示，至少3点
- 所有内容用中文，专业术语准确，通俗易懂
```
3. **大模型调用**：调用豆包/通义千问大模型API，传入组装好的Prompt
4. **结果解析**：解析大模型返回的内容，提取结构化的评分、结论、建议、风险提示
5. **结果存储**：将分析结果存储到MySQL数据库，缓存到Redis
6. **返回结果**：将结构化结果返回给前端展示
### 3.2 持仓分析流程
#### 3.2.1 输入参数
- 用户持仓列表、分析要求
#### 3.2.2 处理步骤
1. **数据采集**：批量采集持仓中所有股票的最新价格、基本面、行情数据
2. **组合计算**：计算持仓总市值、总盈亏、盈亏率、行业分布、风险集中度等指标
3. **Prompt组装**：将持仓数据和组合指标组装成分析Prompt
4. **大模型调用**：调用大模型生成持仓分析报告、优化建议、风险提示
5. **结果返回**：结构化结果返回前端
### 3.3 关注列表监控流程
#### 3.3.1 输入参数
- 用户关注的股票列表、设置的提醒条件（价格阈值、涨跌幅阈值等）
#### 3.3.2 处理步骤
1. **定时采集**：每小时/每天定时采集关注股票的最新行情数据
2. **条件判断**：判断是否触发用户设置的提醒条件
3. **触发通知**：如果触发条件，生成提醒消息，发送给用户
## 4. 输出格式定义（和原设计完全一致，兼容上层接口）
### 4.1 股票分析结果输出（Pydantic模型定义）
```python
class StockAnalysisResult(BaseModel):
    industry_score: float  # 行业评分 0-100
    financial_score: float  # 财务评分 0-100
    technical_score: float  # 技术评分 0-100
    sentiment_score: float  # 舆情评分 0-100
    overall_score: float  # 综合评分 0-100
    analysis_summary: str  # 分析结论，300字以内
    investment_advice: str  # 投资建议：买入/持有/卖出 + 理由
    risk_tips: List[str]  # 风险提示，至少3条
```
### 4.2 持仓分析结果输出
```python
class PortfolioAnalysisResult(BaseModel):
    total_value: float  # 总市值
    total_profit: float  # 总盈亏
    total_profit_rate: float  # 总盈亏率
    industry_distribution: Dict[str, float]  # 行业分布占比
    risk_level: str  # 风险等级：低/中/高
    risk_concentration: str  # 风险集中度说明
    suggestions: List[str]  # 持仓优化建议
    overall_assessment: str  # 综合评价
```
## 5. 大模型配置
### 5.1 推荐模型优先级
1. **优先选择：字节跳动豆包大模型（doubao-seed-2-0-pro）**
   - 优点：中文理解能力强，金融领域效果好，国内访问稳定，延迟低，个人用户有免费额度，完全满足使用需求
   - 费用：免费额度每月100万token，足够个人每天分析10只股票使用；付费版也很便宜，每百万token仅十几元
2. **备选：阿里云通义千问**
   - 优点：性能稳定，金融领域训练充分，国内访问快
3. **本地部署选项：Ollama + Qwen2-7B/14B开源模型**
   - 优点：完全免费，不需要API密钥，所有数据本地处理，隐私安全，不需要联网
   - 要求：电脑配置足够（16G以上内存，最好有Apple Silicon芯片）
### 5.2 模型参数配置
- 温度（temperature）：0.1，降低随机性，确保分析结果稳定
- 最大输出长度：2000 token，足够输出完整的分析报告
- 其他参数：默认即可
## 6. 优化策略
### 6.1 Prompt优化
- 针对A股市场特点优化Prompt，加入A股特有的规则、交易制度说明
- 加入专业的股票分析框架，引导大模型按照专业分析师的思路进行分析
- 加入示例输出，确保大模型返回的格式符合要求
### 6.2 成本优化
- 结果缓存：相同股票的分析结果缓存24小时，避免重复调用大模型，节省成本
- 批量处理：持仓分析、关注列表监控尽量批量处理，减少API调用次数
### 6.3 准确率提升
- 结构化输入：尽量将数据整理成结构化的表格形式，减少大模型理解错误
- 结果校验：对大模型返回的评分进行合理性校验，明显不合理的重新调用
- 人工反馈：支持用户对分析结果进行反馈，根据反馈优化Prompt
## 7. 扩展性设计
1. **多模型支持**：模块化设计，方便添加新的大模型支持，切换不同的AI提供商
2. **自定义分析模板**：支持用户自定义分析要求、Prompt模板，满足个性化的分析需求
3. **回测功能**：未来可以添加历史数据回测功能，验证分析策略的有效性
4. **策略生成**：基于用户的风险偏好，生成个性化的投资策略
