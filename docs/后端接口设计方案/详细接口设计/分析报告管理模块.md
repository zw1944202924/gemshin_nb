# 分析报告管理模块接口设计

## 1. 接口列表

| 接口路径 | 方法 | 功能描述 | 请求参数 | 响应数据 | 权限要求 |
|---------|------|---------|---------|---------|----------|
| `/api/v1/report` | GET | 获取用户分析报告列表 | page: int (默认1，最小值1)<br>page_size: int (默认10，最小值1，最大值50) | `{"total": int, "reports": [{"id": int, "title": str, "stock_code": str, "stock_name": str, "created_at": str, "status": str}]}` | 已登录 |
| `/api/v1/report/{id}` | GET | 获取报告详情 | id: int (路径参数，必填) | `{"id": int, "title": str, "stock_code": str, "stock_name": str, "content": {}, "created_at": str}` | 已登录 |
| `/api/v1/report/{id}/export` | GET | 导出报告 | id: int (路径参数，必填)<br>format: str (必填，"pdf"/"excel") | 文件流 | 已登录 |
| `/api/v1/report/{id}/share` | POST | 分享报告 | id: int (路径参数，必填)<br>email: str (必填，邮箱格式) | `{"message": "报告分享成功"}` | 已登录 |

## 2. 接口详情

### 2.1 获取用户分析报告列表

**接口路径**：`/api/v1/report`
**方法**：GET
**功能描述**：获取用户分析报告列表
**请求参数**：
- page: int (默认1，最小值1)
- page_size: int (默认10，最小值1，最大值50)

**响应数据**：
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "total": 20,
    "reports": [
      {
        "id": 1,

        "stock_code": "600000",
        "stock_name": "浦发银行",
        "created_at": "2026-01-01 12:00:00",
        "status": "已完成"
      },
      {
        "id": 2,
```

### 2.3 导出报告

**接口路径**：`/api/v1/report/{id}/export`
**方法**：GET
**功能描述**：导出报告
**请求参数**：
- id: int (路径参数，必填)
- format: str (必填，"pdf"/"excel")

**响应数据**：
- 文件流（PDF或Excel文件）

### 2.4 分享报告

**接口路径**：`/api/v1/report/{id}/share`
**方法**：POST
**功能描述**：分享报告
**请求参数**：
- id: int (路径参数，必填)
- email: str (必填，邮箱格式)

**响应数据**：
```json
{
  "code": 200,
  "message": "报告分享成功",
  "data": {}
}
```
