# 用户反馈模块接口设计

## 1. 接口列表

| 接口路径 | 方法 | 功能描述 | 请求参数 | 响应数据 | 权限要求 |
|---------|------|---------|---------|---------|----------|
| `/api/v1/feedback` | POST | 提交反馈 | type: str (必填，"bug"/"suggestion"/"question")<br>content: str (必填，长度10-500)<br>contact: str (可选，邮箱或电话) | `{"id": int, "message": "反馈提交成功"}` | 已登录 |
| `/api/v1/feedback` | GET | 获取用户反馈历史 | page: int (默认1，最小值1)<br>page_size: int (默认10，最小值1，最大值50) | `{"total": int, "feedbacks": [{"id": int, "type": str, "content": str, "created_at": str, "status": str, "reply": str}]}` | 已登录 |

## 2. 接口详情

### 2.1 提交反馈

**接口路径**：`/api/v1/feedback`
**方法**：POST
**功能描述**：提交反馈
**请求参数**：
- type: str (必填，"bug"/"suggestion"/"question")
- content: str (必填，长度10-500)
- contact: str (可选，邮箱或电话)

**响应数据**：
```json
{
  "code": 200,
  "message": "反馈提交成功",
  "data": {
    "id": 1,
    "message": "反馈提交成功"
  }
}
```

### 2.2 获取用户反馈历史

**接口路径**：`/api/v1/feedback`
**方法**：GET
**功能描述**：获取用户反馈历史
**请求参数**：
- page: int (默认1，最小值1)
- page_size: int (默认10，最小值1，最大值50)

**响应数据**：
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "total": 10,
    "feedbacks": [
      {
        "id": 1,
        "type": "bug",
        "content": "股票分析页面加载缓慢",
        "created_at": "2026-01-01 10:00:00",
        "status": "已处理",
        "reply": "我们已经优化了页面加载速度，感谢您的反馈"
      },
      {
        "id": 2,
        "type": "suggestion",
        "content": "希望增加更多技术指标分析",
        "created_at": "2026-01-02 14:00:00",
        "status": "处理中",
        "reply": "我们正在考虑增加更多技术指标，敬请期待"
      }
    ]
  }
}
```
