# 用户管理模块接口设计

## 1. 接口列表

| 接口路径 | 方法 | 功能描述 | 请求参数 | 响应数据 | 权限要求 |
|---------|------|---------|---------|---------|----------|
| `/api/v1/auth/register` | POST | 用户注册 | username: str (必填，长度3-20)<br>email: str (必填，邮箱格式)<br>password: str (必填，长度6-20) | `{"user_id": int, "username": str, "email": str}` | 无 |
| `/api/v1/auth/login` | POST | 用户登录 | email: str (必填，邮箱格式)<br>password: str (必填) | `{"access_token": str, "token_type": "Bearer", "user": {"id": int, "username": str, "email": str}}` | 无 |
| `/api/v1/auth/refresh` | POST | 刷新令牌 | refresh_token: str (必填) | `{"access_token": str, "token_type": "Bearer"}` | 无 |
| `/api/v1/user/profile` | GET | 获取用户信息 | 无 | `{"id": int, "username": str, "email": str, "created_at": str}` | 已登录 |
| `/api/v1/user/profile` | PUT | 更新用户信息 | username: str (必填，长度3-20)<br>email: str (必填，邮箱格式) | `{"id": int, "username": str, "email": str}` | 已登录 |
| `/api/v1/user/password` | PUT | 修改密码 | old_password: str (必填)<br>new_password: str (必填，长度6-20) | `{"message": "密码修改成功"}` | 已登录 |
| `/api/v1/admin/user/list` | GET | 获取用户列表 | page: int (默认1，最小值1)<br>page_size: int (默认10，最小值1，最大值50) | `{"total": int, "users": [{"id": int, "username": str, "email": str, "is_admin": bool, "status": str, "created_at": str}]}` | 管理员 |
| `/api/v1/admin/user/{id}/role` | PUT | 修改用户角色 | id: int (路径参数，必填)<br>is_admin: bool (必填) | `{"message": "用户角色修改成功"}` | 管理员 |
| `/api/v1/admin/user/{id}/status` | PUT | 启用/禁用用户 | id: int (路径参数，必填)<br>status: str (必填，"active"/"inactive") | `{"message": "用户状态修改成功"}` | 管理员 |

## 2. 接口详情

### 2.1 用户注册

**接口路径**：`/api/v1/auth/register`
**方法**：POST
**功能描述**：用户注册
**请求参数**：
- username: str (必填，长度3-20)
- email: str (必填，邮箱格式)
- password: str (必填，长度6-20)

**响应数据**：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "user_id": 1,
    "username": "testuser",
    "email": "test@example.com"
  }
}
```

### 2.2 用户登录

**接口路径**：`/api/v1/auth/login`
**方法**：POST
**功能描述**：用户登录
**请求参数**：
- email: str (必填，邮箱格式)
- password: str (必填)

**响应数据**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "Bearer",
    "user": {
      "id": 1,
      "username": "testuser",
      "email": "test@example.com"
    }
  }
}
```

### 2.3 刷新令牌

**接口路径**：`/api/v1/auth/refresh`
**方法**：POST
**功能描述**：刷新令牌
**请求参数**：
- refresh_token: str (必填)

**响应数据**：
```json
{
  "code": 200,
  "message": "令牌刷新成功",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "Bearer"
  }
}
```

### 2.4 获取用户信息

**接口路径**：`/api/v1/user/profile`
**方法**：GET
**功能描述**：获取用户信息
**请求参数**：无
**响应数据**：
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "id": 1,
    "username": "testuser",
    "email": "test@example.com",
    "created_at": "2026-01-01 12:00:00"
  }
}
```

### 2.5 更新用户信息

**接口路径**：`/api/v1/user/profile`
**方法**：PUT
**功能描述**：更新用户信息
**请求参数**：
- username: str (必填，长度3-20)
- email: str (必填，邮箱格式)

**响应数据**：
```json
{
  "code": 200,
  "message": "更新成功",
  "data": {
    "id": 1,
    "username": "newname",
    "email": "newemail@example.com"
  }
}
```

### 2.6 修改密码

**接口路径**：`/api/v1/user/password`
**方法**：PUT
**功能描述**：修改密码
**请求参数**：
- old_password: str (必填)
- new_password: str (必填，长度6-20)

**响应数据**：
```json
{
  "code": 200,
  "message": "密码修改成功",
  "data": {}
}
```

### 2.7 获取用户列表（管理员）

**接口路径**：`/api/v1/admin/user/list`
**方法**：GET
**功能描述**：获取用户列表
**请求参数**：
- page: int (默认1，最小值1)
- page_size: int (默认10，最小值1，最大值50)

**响应数据**：
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "total": 100,
    "users": [
      {
        "id": 1,
        "username": "testuser",
        "email": "test@example.com",
        "is_admin": false,
        "status": "active",
        "created_at": "2026-01-01 12:00:00"
      }
    ]
  }
}
```

### 2.8 修改用户角色（管理员）

**接口路径**：`/api/v1/admin/user/{id}/role`
**方法**：PUT
**功能描述**：修改用户角色
**请求参数**：
- id: int (路径参数，必填)
- is_admin: bool (必填)

**响应数据**：
```json
{
  "code": 200,
  "message": "用户角色修改成功",
  "data": {}
}
```

### 2.9 启用/禁用用户（管理员）

**接口路径**：`/api/v1/admin/user/{id}/status`
**方法**：PUT
**功能描述**：启用/禁用用户
**请求参数**：
- id: int (路径参数，必填)
- status: str (必填，"active"/"inactive")

**响应数据**：
```json
{
  "code": 200,
  "message": "用户状态修改成功",
  "data": {}
}
```
