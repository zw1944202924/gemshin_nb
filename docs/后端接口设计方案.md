# 后端接口设计方案

## 1. 接口设计原则

- **RESTful API设计**：遵循RESTful API设计规范，使用HTTP方法（GET, POST, PUT, DELETE）表示操作类型
- **模块化设计**：按照系统模块划分API路由，使用Flask蓝图(Blueprint)组织API路由
- **参数验证**：使用Flask-WTF等库验证请求参数的合法性
- **响应格式统一**：所有API返回JSON格式数据，包含状态码、消息和数据
- **错误处理**：统一处理错误，返回标准错误响应
- **认证授权**：使用JWT实现用户认证和权限控制

## 2. 通用响应格式

```json
{
  "code": 200,              // 状态码
  "message": "成功",        // 消息
  "data": {}                // 数据
}
```

#### 3.8.1 数据导入格式说明

**持仓数据 (portfolio) 格式**：
- **Excel/CSV字段**：
  - `code`：股票代码（必填）
  - `name`：股票名称（必填）
  - `quantity`：持有数量（必填，整数）
  - `cost_price`：成本价（必填，数值）

**交易记录 (transaction) 格式**：
- **Excel/CSV字段**：
  - `code`：股票代码（必填）
  - `name`：股票名称（必填）
  - `type`：交易类型（必填，"buy"或"sell"）
  - `quantity`：交易数量（必填，整数）
  - `price`：交易价格（必填，数值）
  - `date`：交易日期（必填，YYYY-MM-DD格式）

**关注列表 (watchlist) 格式**：
- **Excel/CSV字段**：
  - `code`：股票代码（必填）
  - `name`：股票名称（必填）
  - `target_price`：目标价格（必填，数值）

#### 3.8.2 数据导入处理流程

1. **文件上传**：用户通过接口上传Excel或CSV文件
2. **文件验证**：验证文件格式和大小
3. **数据解析**：使用pandas库解析文件数据
4. **数据验证**：验证每一行数据的合法性
5. **数据导入**：将验证通过的数据导入数据库
6. **错误处理**：记录导入失败的数据和错误原因
7. **结果返回**：返回导入成功和失败的数量，以及错误详情

#### 3.8.3 数据导出处理流程

1. **参数验证**：验证导出类型和格式参数
2. **数据查询**：从数据库查询用户的相关数据
3. **数据处理**：将数据转换为适合导出的格式
4. **文件生成**：根据指定格式生成Excel或CSV文件
5. **文件返回**：将生成的文件作为响应返回给用户

#### 3.8.4 导入/导出示例

**导入示例**：
```bash
# 使用curl导入持仓数据
curl -X POST "http://localhost:5000/api/v1/data/import" \
  -H "Authorization: Bearer {access_token}" \
  -F "file=@portfolio.xlsx" \
  -F "data_type=portfolio"
```

**导出示例**：

## 4. 认证与授权

### 4.1 JWT认证流程

1. 用户通过 `/api/v1/auth/login` 接口获取访问令牌
2. 前端在后续请求的 `Authorization` 头中携带令牌：`Bearer {access_token}`
3. 后端验证令牌的有效性
4. 对于需要权限的接口，验证用户是否具有相应权限

### 4.2 权限控制

- **无权限**：无需登录即可访问的接口，如登录、注册等
- **已登录**：需要用户登录才能访问的接口，如获取用户信息、查看持仓等
- **管理员**：需要管理员权限才能访问的接口，如AI模型管理等

### 4.3 管理员权限实现

1. **管理员账户创建**：
   - 系统初始化时创建默认管理员账户
   - 管理员可以通过管理界面创建新的管理员账户

2. **权限验证实现**：
   - 使用Flask的装饰器实现权限验证
   - 在用户模型中添加 `is_admin` 字段标识管理员身份
   - 对于需要管理员权限的接口，使用 `@admin_required` 装饰器

3. **权限管理接口**：
   - `/api/v1/admin/user/list`：获取用户列表
   - `/api/v1/admin/user/{id}/role`：修改用户角色
   - `/api/v1/admin/user/{id}/status`：启用/禁用用户

### 4.4 权限验证示例

```python
from flask_jwt_extended import jwt_required, get_jwt_identity
from functools import wraps

def admin_required(f):
    @wraps(f)
    @jwt_required()
    def decorated_function(*args, **kwargs):
        current_user = get_jwt_identity()
        user = User.query.get(current_user)
        if not user or not user.is_admin:
            return {"code": 403, "message": "禁止访问", "data": {}}, 403
        return f(*args, **kwargs)
    return decorated_function

# 使用示例

@admin_required

    pass
```

## 6. 接口安全

- **HTTPS**：使用HTTPS协议加密传输数据
- **JWT令牌**：使用JWT进行身份验证，设置合理的过期时间
- **参数验证**：对所有请求参数进行验证，防止注入攻击
- **CORS**：设置合理的CORS策略，防止跨站请求伪造
- **速率限制**：对API请求进行速率限制，防止暴力攻击
- **敏感数据加密**：对敏感数据进行加密存储

## 7. 性能优化

- **缓存**：使用Redis缓存热点数据，减少数据库查询
- **异步处理**：使用Celery处理耗时的任务，如数据采集和模型分析
- **分页**：对列表接口实现分页，减少数据传输量
- **批量操作**：支持批量操作，减少API调用次数
- **索引优化**：对数据库查询进行索引优化

## 11. 文档维护

### 11.1 API文档生成

- **使用Swagger**：集成Flask-RESTPlus或Flask-RESTX库生成Swagger文档
- **配置方法**：
  1. 安装依赖：`pip install flask-restx`
  2. 在应用中初始化Swagger：
     ```python
     from flask_restx import Api
     
     api = Api(
         app,
         version='1.0',
         title='AI股票分析助手API',
         description='AI股票分析助手后端API文档',
         doc='/api/docs'
     )
     ```
  3. 为每个路由添加文档注释：
     ```python
     from flask_restx import Resource, fields
     
     auth_ns = api.namespace('auth', description='认证相关接口')
     
     register_model = auth_ns.model('Register', {
         'username': fields.String(required=True, description='用户名', min_length=3, max_length=20),
         'email': fields.String(required=True, description='邮箱', example='user@example.com'),
         'password': fields.String(required=True, description='密码', min_length=6, max_length=20)
     })
     
     @auth_ns.route('/register')
     class Register(Resource):
         @auth_ns.expect(register_model)
         @auth_ns.response(200, '注册成功')
         @auth_ns.response(400, '请求参数错误')
         def post(self):
             """用户注册"""
             # 实现注册逻辑
             pass
     ```

### 11.4 示例代码

- **前端调用示例**：
  ```javascript
  // 使用Axios调用登录接口
  import axios from 'axios';
  
  async function login(email, password) {
    try {
      const response = await axios.post('/api/v1/auth/login', {
        email: email,
        password: password
      });
      return response.data;
    } catch (error) {
      console.error('登录失败:', error);
      throw error;
    }
  }
  
  // 使用JWT令牌调用需要认证的接口
  async function getPortfolio() {
    try {
      const token = localStorage.getItem('access_token');
      const response = await axios.get('/api/v1/portfolio', {
        headers: {
          'Authorization': `Bearer ${token}`
        }
      });
      return response.data;
    } catch (error) {
      console.error('获取持仓失败:', error);
      throw error;
    }
  }
  ```

- **后端调用示例**：
  ```python
  import requests
  
  # 登录获取令牌
  def login(email, password):
      response = requests.post('http://localhost:5000/api/v1/auth/login', {
          'email': email,
          'password': password
      })
      return response.json()
  
  # 使用令牌调用接口
  def get_stock_analysis(stock_code, analysis_type, token):
      headers = {
          'Authorization': f'Bearer {token}'
      }
      response = requests.post(
          f'http://localhost:5000/api/v1/stock/{stock_code}/analysis',
          json={'analysis_type': analysis_type},
          headers=headers
      )
      return response.json()
  ```
