# 权限管理

## 9.1 角色定义

### 9.1.1 系统角色
| 角色 | 描述 | 权限范围 |
|------|------|----------|
| **超级管理员** | 系统最高权限 | 所有功能，包括用户管理、权限配置等 |
| **管理员** | 管理权限 | 除系统设置外的所有功能 |
| **分析师** | 分析权限 | 股票分析、报告管理、板块分析等 |
| **普通用户** | 基础权限 | 个人持仓管理、关注列表、通知等 |
| **访客** | 只读权限 | 只能查看公开信息，如市场概览、板块分析等 |

### 9.1.2 权限级别
- **Level 0**：无权限（未登录）
- **Level 1**：只读权限（访客）
- **Level 2**：基础操作权限（普通用户）
- **Level 3**：分析操作权限（分析师）
- **Level 4**：管理权限（管理员）
- **Level 5**：系统权限（超级管理员）

## 9.4 权限验证机制

### 9.4.1 前端实现
1. **路由守卫**：使用Vue Router的路由守卫，在路由切换时验证权限
2. **权限指令**：自定义Vue指令，控制页面元素的显示/隐藏
3. **权限钩子**：在组件挂载前验证权限
4. **状态管理**：在Pinia中存储用户权限信息

### 9.4.2 后端实现
1. **API权限验证**：每个API接口都需要验证用户权限
2. **JWT令牌**：使用JWT令牌存储用户权限信息
3. **权限中间件**：在后端路由中使用权限中间件验证权限
